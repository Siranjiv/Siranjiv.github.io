Cemented Dreams

February 2026 Dev Blog

Siranjiv Sivapalan | Weekly Engineering Notes

This month focused on gameplay responsiveness and a new dynamic environment system. Work included implementing Coyote Time for jump forgiveness, fixing edge-overhang behavior, and completing the C++ Hive Platform system with designer-facing controls and debugging tools.

February Snapshot

  • Implemented Coyote Time to improve jump responsiveness and platforming feel.
  • Fixed edge-overhang/slide behavior to improve movement control and reduce unintended falls.
  • Started and then completed the Hive Platform system in C++.
  • Added robust designer controls and debug visualization for tuning dynamic hex growth behavior.

Week 1

Coyote Time

This week I was assigned a smaller gameplay task while the design team discussed upcoming implementations. My director asked me to implement Coyote Time for the player jump mechanic.

I added a short grace window after leaving a platform, allowing jump input to still register for a brief duration. This improves movement responsiveness and makes platforming feel more forgiving, especially in fast-paced or precision scenarios.

Overall, this week focused on core player-control polish while larger design decisions were still being finalized.

Week 2

Hive Platform Start

I first fixed a gameplay bug where the character could unintentionally overhang or slide off edges. The fix ensures the character properly stops at edges during movement, improving control and preventing accidental falls.

After the fix, I started development of the Hive Platform system. Unlike earlier transformation systems, this one uses hexagonal tiles that grow upward from below the ground as the player approaches.

  • Each tile has a fixed root beneath the surface.
  • Tiles do not translate through space; geometry extrudes vertically.
  • The environment appears to build itself in response to player movement.

Week 3

Midterms + Iteration

This week I was given time by my game director to prepare for midterm exams. After completing exams, I resumed work on Hive Platform development.

I continued refining core hex-platform behavior, with additional focus on player-driven response consistency and stable growth patterns from beneath the ground.

Week 4

Hive Platform Finalization

This week I completed the Hive Platform system. The full implementation is in C++, enabling strong performance, detailed control, and scalability for dynamic environment generation.

As the player or camera approaches, hex tiles extrude upward from fixed underground roots to form a real-time traversable surface. Designers can tune behavior through the following parameters:

  • TravelDistance: vertical extension from lowest to fully visible state.
  • StartOffset: initial hidden depth below ground.
  • ActivationRadius: distance where growth starts.
  • FullExtendRadius: distance where tiles are fully extended.
  • InterpSpeed: transition speed, from smooth to snappy behavior.
  • Reference Source: activation reference based on camera or character.

I also added debug outlines and radius indicators to help designers understand runtime behavior and tune values quickly.

Note: The tutorial video is titled "Hex Platform", but the correct system name is "Hive Platform".
