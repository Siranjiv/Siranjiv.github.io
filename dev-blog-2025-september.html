Cemented Dreams

September 2025 Dev Blog

Siranjiv Sivapalan | Weekly Engineering Notes

This month focused on two major areas: building a reusable glitch material system for stylized visual effects, and implementing a movement transformation gameplay system in C++ with spline-based motion, optional acceleration/deceleration, and player interaction support.

September Snapshot

  • Prototyped and shipped a first full version of a customizable glitch material.
  • Expanded the glitch system into multiple variants: geometry and UV-based effects.
  • Built a core gameplay movement transformation system in C++ for spline-following objects.
  • Added designer-facing controls for speed, visual strength, interpolation, and movement behavior.

Week 1

Glitch Prototype

I worked on the early-stage glitch material requested by the director. This first pass focused on establishing the core visual behavior and stress-testing how the effect performs across different scene conditions. At this stage, the material was intentionally iterative, with ongoing parameter experimentation to improve quality and flexibility.

Week 2

First Full Version

I completed the first full version of the glitch material and moved from prototype behavior to controlled, production-usable tuning. The system now includes a robust set of exposed controls so the effect can be adjusted quickly for different scenarios.

  • Speed Threshold
  • Max Glitch Strength
  • Min Channel Offset and Max Channel Offset
  • Flicker Strength
  • Lerp Speed

Week 3

Glitch Variants

I expanded the glitch pipeline into multiple material variants to support a wider range of artistic direction and gameplay contexts. The focus was building a reusable effect family, not just one isolated shader behavior.

  • Geometry Glitch V1 and V2
  • UV Glitch (Static)
  • UV Glitch (Dynamic)
  • UV Glitch (Static) with heavier stutter behavior
  • UV Glitch (Dynamic) with field depth, strength, and glitch color controls

Week 4

Spline Movement System (C++)

This week I shifted from material work to a core gameplay system requested by the director: a movement transformation system that moves objects along a customizable spline path. I developed it in a separate project first for a clean workflow before integration into the main game.

  • Smooth start-to-end spline traversal with adjustable MoveSpeed.
  • Reliable stop behavior at spline end for controlled gameplay timing.
  • Optional acceleration/deceleration toggle (Use Acceleration Deceleration) for more natural motion.
  • Player-on-platform support while the object is moving.
  • Mesh swapping support so designers can replace the default cube with project assets.
  • Implemented in C++ rather than Blueprint for stronger performance, control, and scalability.
